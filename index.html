The fast path — do this tonight

Chat AI → OverleafBrowser only · ~15 min first time
You’ll need: any chat-based AI — Ask Sage (the DoD-authorized, CUI-approved one), or ChatGPT / Claude / Gemini for non-sensitive memos — and a free Overleaf account. Plus the starter kit above (just unzip it). For sensitive (CUI) memos, use asksage.ai (Army CAC users sign in at chat.genai.army.mil). For routine memos, any chatbot you already have works.
Part A · Write it (in the chat)
  1. Unzip the starter kit. (Nothing goes to the AI — you’ll only upload two files to Overleaf later.)
  2. Open your AI (Ask Sage, ChatGPT, Claude…) and start a new chat. Pick the most capable model offered (the newest one, usually at the top).
  3. Open sample-prompt.md from the kit, copy the whole thing, and paste it into the chat. The memo format is built into the prompt — no file uploads needed.
  4. Fill the [brackets] with your memo facts, then send.
  5. The AI replies with the LaTeX. Click Copy — but copy only the code: it must start with \documentclass. Delete any ``` lines or sentences it adds.
Part B · Get the PDF (Overleaf)
  1. Go to overleaf.com → sign up free → New Project → Blank Project.
  2. Click Upload and add armymemo.cls and DODb1.pdf from the kit.
  3. Paste the copied LaTeX into the main file (replace what’s there).
  4. Important: Menu → Compiler → LuaLaTeX (not the default). Click Recompile.
  5. Click Download PDF. That’s your memo.
  6. Blanks? Anywhere the AI lacked a fact shows an underline. Find the matching \rule{…} in the source, type your text over it, Recompile.
Heads up — expect a little fiddling. This is AI + LaTeX, not a one-click button. The first time, plan on some trial and error: sometimes the AI wraps its answer in ``` marks or adds a sentence (delete those); now and then it returns code that won’t compile (paste the error message back into the chat and ask it to fix it); and you’ll tweak wording and fill in blanks before it’s right. It’s still far faster than formatting by hand.

Two non-negotiables — these matter more than any step above

1. Mind where the words go. Ask Sage is a government, CUI-approved system — fine for sensitive memo content. Overleaf and commercial chatbots are not. Never paste CUI, PHI, or real PII into them. If your memo contains CUI, skip Overleaf — compile it on your own computer instead (below), so nothing leaves your machine.

2. You sign it, you own it. AI invents names, dates, and regulation numbers with total confidence — a wrong reg number looks exactly as authoritative as a right one. The sample prompt tells it to leave blanks instead of guessing. Read every line before it goes out; “the AI wrote it” is not a defense.

Optional — when you’re ready to go faster

OptionalThe fast path above is a complete, permanent solution — many people never need more. If you make memos often, here are two setups that remove the copy-paste.

First, what you were actually doing. That “block of code” is LaTeX (say “LAY-tek”), a typesetting system. A compiler turns it into the finished PDF — that’s what Overleaf did. The formatting all comes from the class file armymemo.cls: it encodes the AR 25-50 rules, so you and the AI supply the words and it handles the layout. The two upgrades below just move the writing and the compiling onto your own computer so there’s no carrying files between two websites.

Claude Cowork

Desktop agent · writes the file for you
Need: the Claude desktop app (Cowork, on any paid plan — Pro or higher).
  1. Put the starter-kit folder in your Documents folder.
  2. Open the app, click Cowork, point it at that folder, grant access.
  3. Tell it: “Using the armymemo template here, write an MFR about [your facts]. Save it as a new .tex file. Don’t make anything up — leave blanks.”
  4. It writes the file for you. For the PDF, hand the file to Overleaf, or compile locally if you’ve installed LaTeX (below).

Claude Code in VS Code

The way I do it in class · most setup, least effort after
Need: VS Code, Claude Code, and LaTeX installed (below).
  1. Open a folder containing armymemo.cls + DODb1.pdf in VS Code; start Claude Code.
  2. Ask in plain English: “Using armymemo.cls here, write an MFR about [facts], leave blanks for anything missing, then compile twice with lualatex and open the PDF.”
  3. It writes the memo, compiles it, and opens the PDF — no copy-paste, and nothing leaves your machine.
  4. Iterate by talking. (In class I use a custom /memo skill that also auto-loads the regs; you can build one once you’re comfortable.)
Compiling on your own computer (needed for both upgrades, and for any CUI memo): install MacTeX (Mac) or TeX Live (Windows/Linux) — one time, a few GB. Keep armymemo.cls and DODb1.pdf next to your .tex file and run, twice, in a terminal: lualatex memo.tex && lualatex memo.tex. Because it’s all on your machine, this path is safe for CUI. (The memo uses Times New Roman + Arial; Mac and Overleaf have them, a bare Linux box may need the msttcorefonts package first.)

What’s in the starter kit

  • sample-prompt.md — the ready-to-paste prompt for Part A (no uploads).
  • armymemo.cls — the rulebook (all AR 25-50 formatting). Upload to Overleaf.
  • DODb1.pdf — the DoD seal. Upload to Overleaf too.
  • mfr-template-EXAMPLE-output.pdf — what a finished memo looks like.
  • mfr-template.tex — a blank fill-in memo, if you’d rather start by hand.
  • README.txt — the same steps, in the folder.
  • digsig.sty — optional; ignore unless you add a digital signature.
